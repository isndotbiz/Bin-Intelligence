Exploit Types Explained
  • Card-Not-Present: Fraud in online purchases where the physical card is absent. Fraudsters use stolen data for test transactions before larger purchases.
  • Cross-Border: Exploits cards in countries different from issuance, targeting regions with weaker controls where banks have limited visibility.
  • False-Positive CVV: Banks fail to properly validate security codes, approving transactions with any CVV. Common in smaller banks with outdated systems.
  • No Auto-3DS: Cards lacking frictionless authentication that require disruptive verification steps, leading to higher fraud and abandonment rates.

3DS Support Explained
  • No 3DS: Cards lacking 3D Secure protection are highly vulnerable to fraud, with no additional verification during checkout.
  • 3DS v1: First-generation protocol requiring manual verification, often causing checkout friction and abandonment.
  • 3DS v2: Enhanced protocol with risk-based authentication, improved user experience, and stronger fraud prevention.
  • Auto 3DS: Seamless authentication without user intervention, providing maximum security with minimal friction.
BIN Classification Methodology

Data Sources: BINs are classified based on two primary sources: fraud feeds from monitored networks and verified data from Neutrino API. Classification begins with pattern identification in known fraud incidents. BINs are classified by exploit type based on patterns observed in these feeds and authenticated Neutrino API data.

Classification Process: Our system uses a multi-stage analysis pipeline that first identifies exploited BINs, then verifies their characteristics (issuing bank, country, card type) with Neutrino API. The exploit type is determined by analyzing transaction patterns, authentication failures, and geographic anomalies.

Patch Status: A BIN's patch status is determined primarily by its 3DS support level. Cards with 3DS v2 or Auto-3DS are labeled "Patched," as they have modern authentication capabilities. BINs with no 3DS or only 3DS v1 are labeled "Exploitable" and should receive enhanced monitoring. Cards without 3DS support are significantly more vulnerable to attack vectors and should be prioritized for additional security measures.

Risk Scoring: BINs are assigned risk scores based on a weighted combination of exploit frequency, patch status, and historical fraud rates. Higher scores indicate greater vulnerability and should trigger additional verification measures during checkout.
