BIN Classification Methodology
Card-Not-Present Fraud

BINs vulnerable to e-commerce fraud where physical cards aren't required. These transactions rely on card details only, making them susceptible to data theft and unauthorized use in online environments.

False-Positive CVV

BINs where the Card Verification Value (CVV) validation is weak or bypassed. These cards accept any CVV value, allowing fraudsters to complete online purchases without knowing the real security code.

No Auto-3DS

BINs lacking automatic 3D Secure authentication, which is an additional security layer for online transactions. Without this protection, cardholders aren't prompted for verification, increasing fraud vulnerability.

Patch Status

Patched BINs have implemented necessary security features including 3DS support. Exploitable BINs lack critical security controls, making them vulnerable to e-commerce fraud methods.
